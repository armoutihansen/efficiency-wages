# Efficiency Wages with Motivated Agents

This repository is a replication package for **"Efficiency Wages with Motivated Agents"** and its online appendix.

Publication: https://www.sciencedirect.com/science/article/pii/S0899825624000307?via%3Dihub

The published paper and online appendix are not included in this repository. The publication link above is the source of truth for the article and supplement.

## Three Steps

1. Install the requirements:
   - `uv`: https://docs.astral.sh/uv/
   - Stata 17 or newer, if you want to rerun the regression tables.

2. Run the replication from the repository root:

```bash
uv run python replicate.py
```

3. Open the generated report:

```text
outputs/replication_report.html
```

The report shows the detected operating system, Python version, Stata executable and version, verification checks, generated tables, generated figures, and the map from paper artifacts to repository outputs.

## Stata Setup

Full replication requires a licensed Stata installation because the paper and appendix regression tables are generated by Stata `.do` files.

The replication runner works on macOS, Windows, and Linux when Stata is available on `PATH` or when `STATA_PATH` points to the Stata executable.

Examples:

```bash
STATA_PATH="/Applications/Stata/StataSE.app/Contents/MacOS/stata-se" uv run python replicate.py
```

```powershell
$env:STATA_PATH="C:\Program Files\Stata19\StataSE-64.exe"
uv run python replicate.py
```

To check setup without rebuilding results:

```bash
uv run python replicate.py --check-setup
```

To rebuild the Python outputs and report without rerunning Stata:

```bash
uv run python replicate.py --skip-stata
```

In `--skip-stata` mode, the report marks Stata tables as not rerun.

## What Gets Rebuilt

- `data/derived/`: assembled analysis datasets.
- `outputs/tables/`: generated main-paper and appendix table data.
- `outputs/figures/`: generated main-paper and appendix figures.
- `outputs/correspondence.md`: map from paper artifacts to generated outputs.
- `outputs/verification_report.json`: machine-readable verification checks.
- `outputs/replication_report.html`: researcher-facing verification report.

The committed outputs are included so readers can inspect the regenerated results without running the full pipeline.

## Troubleshooting

- If Stata is not found, rerun with `STATA_PATH` set to the full Stata executable path.
- If Stata is older than version 17, install or point to a newer Stata executable.
- If `uv` is not found, install it from https://docs.astral.sh/uv/.
- If verification fails, open `outputs/replication_report.html` and `outputs/verification_report.json` to see which check failed.
- If you only want to inspect Python-generated data and figures, run `uv run python replicate.py --skip-stata`.
