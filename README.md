# Efficiency Wages with Motivated Agents

This repository is a replication package for **"Efficiency Wages with Motivated Agents"** and its online appendix.

Publication: https://www.sciencedirect.com/science/article/pii/S0899825624000307?via%3Dihub

## Layout

- `data/raw/`: treatment-level CSV inputs used by the replication pipeline.
- `data/derived/`: generated analysis samples.
- `src/data/`: data assembly.
- `src/analysis/`: figure scripts and Stata table scripts.
- `src/verification/`: smoke, numeric, artifact, and environment checks.
- `outputs/tables/`: generated regression and summary tables.
- `outputs/figures/`: generated paper and appendix figures.
- `outputs/correspondence.md`: publication artifact checklist.

## Requirements

- `uv` for the Python environment and command execution.
- Stata 19 with a command-line executable available at `/Applications/Stata/StataSE.app/Contents/MacOS/stata-se`, `/Applications/Stata/StataMP.app/Contents/MacOS/stata-mp`, `/Applications/Stata/StataBE.app/Contents/MacOS/stata-be`, or on `PATH` as `stata-se`, `stata-mp`, `stata-be`, or `stata`.

The Stata regressions are run through the Stata command-line interface. The pipeline does not use `pystata`.

## Rebuild

Run the full scripted pipeline from the repository root:

```bash
make all
```

The same pipeline is also available as:

```bash
uv run python main.py
```

Equivalent individual steps:

```bash
uv run python -m src.data.assemble
uv run python -m src.analysis.figures
uv run python -m src.analysis.run_stata_tables
uv run python -m src.verification.correspondence
uv run python -m src.verification.checks
```

Step context:

- `src.data.assemble` reads the treatment-level CSV inputs, adds treatment labels and role indicators, and writes the common wide and long analysis datasets in `data/derived/`.
- `src.analysis.figures` reads `data/derived/` and regenerates the currently scripted main and appendix figures in `outputs/figures/`.
- `src.analysis.run_stata_tables` writes the supplement summary statistics and calls Stata 19 CLI for `src/analysis/tables_main.do` and `src/analysis/tables_appendix.do`. These regenerate main-paper Tables 2-4 into `outputs/tables/main_tables_results.csv`, supplement Table A.1 into `outputs/tables/appendix_table_a1_summary_statistics.csv`, and supplement Tables A.2-A.9 into `outputs/tables/appendix_tables_results.csv`.
- `src.verification.correspondence` writes the artifact checklist in `outputs/correspondence.md`.
- `src.verification.checks` verifies treatment counts, headline numeric results, generated artifacts, and Stata CLI availability.

## Current Replication Status

The implemented checks verify:

- Treatment counts: GE/prosocial `n=190`, neutral `n=90`, efficiency `n=120`.
- Offered wage means: GE `25.574`, prosocial `35.771`.
- Offered-wage treatment coefficient: `10.196`.
- Average real profit-maximizing wages: GE `45`, prosocial `40`.
- Supplement Table A.1 summary statistics for age, gender, study subject, and observations.
- Representative published coefficients from supplement Tables A.2-A.9.
- Generation of main Figures 2-4 and supplement Figures A.1-A.8.
- Stata 19 CLI availability.

See `outputs/correspondence.md` after running the pipeline for the remaining parity checklist.

The checks are not a full cell-by-cell assertion for every published table. They cover the sample structure, headline main-paper results, Table A.1 summary statistics, and representative coefficients from each supplement regression table. The regenerated figures are checked for successful artifact creation; visual parity remains a manual review step.

## Reading Outputs

Regression outputs are long-form CSV files. Each row is one coefficient or constant from a specific published table and model column.

Examples:

```bash
uv run python -c "import pandas as pd; r = pd.read_csv('outputs/tables/main_tables_results.csv'); print(r[(r.table_id == 'Table 4') & (r.model == 1)])"
```

```bash
uv run python -c "import pandas as pd; r = pd.read_csv('outputs/tables/appendix_tables_results.csv'); print(r[(r.table_id == 'Table A.6') & (r.model.astype(str) == '2')])"
```

The correspondence checklist in `outputs/correspondence.md` maps each paper and online-appendix artifact to its script and generated output path.

The figure script also writes `outputs/tables/wage_summary.csv`, the summary data used by the wage-comparison figures.

## Supplement Table Status

The current scripted pipeline implements the supplement tables:

- Table A.1 is generated from `data/derived/analysis_sample.csv` by `src/analysis/tables_appendix.py`.
- Tables A.2-A.3 are random-effects regressions of real and expected effort on wage for each treatment, generated by `src/analysis/tables_appendix.do`.
- Tables A.4-A.5 are acceptance robustness regressions with neutral and efficiency treatments added.
- Tables A.6-A.7 are effort robustness regressions with neutral and efficiency treatments added.
- Tables A.8-A.9 are offered-wage OLS robustness regressions with neutral and efficiency treatments added.

The verification script checks representative published coefficients from each supplement regression table and the full treatment-level summary statistics table.
